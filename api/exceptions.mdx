## Overview

The Mention Python SDK defines a hierarchy of exception classes to handle different types of errors that may occur when interacting with the API.

## Exception Hierarchy

```
MentionError (base exception)
├── MentionAPIError
│   ├── MentionAuthError (401, 403)
│   ├── MentionNotFoundError (404)
│   └── MentionRateLimitError (429)
├── MentionValidationError
└── MentionConnectionError
```

## Exception Classes

### MentionError

Base exception for all Mention API errors. All other exceptions inherit from this class.

**Attributes:**

- `message` (str): The error message
- `details` (dict): Optional dictionary containing additional error details

**Example:**

```python
from mention import MentionError

try:
    # API operation
    pass
except MentionError as e:
    print(f"Error: {e.message}")
    print(f"Details: {e.details}")
```

### MentionAPIError

Exception raised when the API returns an error response. Inherits from `MentionError`.

**Attributes:**

- `message` (str): The error message
- `status_code` (int): HTTP status code returned by the API
- `response_body` (dict | str | None): The response body from the API
- `details` (dict): Contains `status_code` and `response` keys

**Raised when:**
- The API returns a non-2xx HTTP status code (except for specific errors handled by subclasses)

**Example:**

```python
from mention import MentionClient, MentionAPIError

client = MentionClient(api_key="your_api_key")

try:
    alert = client.alerts.get(alert_id="invalid_id")
except MentionAPIError as e:
    print(f"API error: {e.message}")
    print(f"Status code: {e.status_code}")
    print(f"Response: {e.response_body}")
```

### MentionAuthError

Exception raised for authentication or authorization errors. Inherits from `MentionAPIError`.

**HTTP Status Codes:** 401 (Unauthorized), 403 (Forbidden)

**Attributes:**

Inherits all attributes from `MentionAPIError`:
- `message` (str): The error message
- `status_code` (int): 401 or 403
- `response_body` (dict | str | None): The response body from the API

**Raised when:**
- Invalid API key is provided (401)
- API key lacks permission for the requested resource (403)

**Example:**

```python
from mention import MentionClient, MentionAuthError

client = MentionClient(api_key="invalid_key")

try:
    alerts = client.alerts.list()
except MentionAuthError as e:
    print(f"Authentication failed: {e.message}")
    print(f"Status code: {e.status_code}")
    # Handle re-authentication or check permissions
```

### MentionNotFoundError

Exception raised when a resource is not found. Inherits from `MentionAPIError`.

**HTTP Status Code:** 404 (Not Found)

**Attributes:**

Inherits all attributes from `MentionAPIError`:
- `message` (str): The error message
- `status_code` (int): 404
- `response_body` (dict | str | None): The response body from the API

**Raised when:**
- The requested alert, mention, or other resource does not exist
- The resource ID is invalid or has been deleted

**Example:**

```python
from mention import MentionClient, MentionNotFoundError

client = MentionClient(api_key="your_api_key")

try:
    alert = client.alerts.get(alert_id="nonexistent_id")
except MentionNotFoundError as e:
    print(f"Resource not found: {e.message}")
    # Handle missing resource (e.g., create it or use a different ID)
```

### MentionRateLimitError

Exception raised when the API rate limit is exceeded. Inherits from `MentionAPIError`.

**HTTP Status Code:** 429 (Too Many Requests)

**Attributes:**

Inherits all attributes from `MentionAPIError`, plus:
- `message` (str): The error message
- `status_code` (int): 429
- `response_body` (dict | str | None): The response body from the API
- `retry_after` (int | None): Number of seconds to wait before retrying (if provided by API)

**Raised when:**
- Too many requests are sent to the API within a given time period
- The account's rate limit has been exceeded

**Example:**

```python
import time
from mention import MentionClient, MentionRateLimitError

client = MentionClient(api_key="your_api_key")

try:
    mentions = client.mentions.list(alert_id="alert_id")
except MentionRateLimitError as e:
    print(f"Rate limit exceeded: {e.message}")
    if e.retry_after:
        print(f"Retry after {e.retry_after} seconds")
        time.sleep(e.retry_after)
        # Retry the request
    else:
        print("Rate limit exceeded, please try again later")
```

### MentionValidationError

Exception raised for request validation errors. Inherits from `MentionError`.

**Attributes:**

Inherits all attributes from `MentionError`:
- `message` (str): The error message describing the validation issue
- `details` (dict): Optional details about the validation failure

**Raised when:**
- Required parameters are missing
- Parameters have invalid types or values
- Request data fails client-side validation

**Example:**

```python
from mention import MentionClient, MentionValidationError

client = MentionClient(api_key="your_api_key")

try:
    # Invalid parameter type
    alert = client.alerts.update(alert_id=123)  # Should be string
except MentionValidationError as e:
    print(f"Validation error: {e.message}")
    print(f"Details: {e.details}")
    # Fix the validation issue and retry
```

### MentionConnectionError

Exception raised for network or connection errors. Inherits from `MentionError`.

**Attributes:**

Inherits all attributes from `MentionError`:
- `message` (str): The error message describing the connection issue
- `details` (dict): Optional details about the connection failure

**Raised when:**
- Unable to establish a connection to the API
- Network timeout occurs
- DNS resolution fails
- SSL/TLS errors occur

**Example:**

```python
from mention import MentionClient, MentionConnectionError

client = MentionClient(api_key="your_api_key")

try:
    alerts = client.alerts.list()
except MentionConnectionError as e:
    print(f"Connection error: {e.message}")
    print("Please check your internet connection and try again")

```

## Error Handling Best Practices

### Catch Specific Exceptions

Handle specific exceptions to provide appropriate error handling for different scenarios:

```python
from mention import (
    MentionClient,
    MentionAuthError,
    MentionNotFoundError,
    MentionRateLimitError,
    MentionConnectionError,
    MentionAPIError,
    MentionError
)

client = MentionClient(api_key="your_api_key")

try:
    alert = client.alerts.get(alert_id="alert_id")
except MentionAuthError:
    print("Authentication failed. Please check your API key.")
except MentionNotFoundError:
    print("Alert not found. Please check the alert ID.")
except MentionRateLimitError as e:
    if e.retry_after:
        print(f"Rate limited. Retry after {e.retry_after} seconds.")
except MentionConnectionError:
    print("Connection failed. Please check your network.")
except MentionAPIError as e:
    print(f"API error: {e.status_code} - {e.message}")
except MentionError as e:
    print(f"Mention error: {e.message}")
```

### Implement Retry Logic

For transient errors like rate limits and connection issues:

```python
import time
from mention import MentionClient, MentionRateLimitError, MentionConnectionError

def get_alert_with_retry(client, alert_id, max_retries=3):
    for attempt in range(max_retries):
        try:
            return client.alerts.get(alert_id=alert_id)
        except MentionRateLimitError as e:
            if e.retry_after:
                time.sleep(e.retry_after)
            else:
                time.sleep(2 ** attempt)  # Exponential backoff
        except MentionConnectionError:
            if attempt < max_retries - 1:
                time.sleep(2 ** attempt)
            else:
                raise
    raise Exception("Max retries exceeded")
```

### Log Error Details

Capture error details for debugging and monitoring:

```python
import logging
from mention import MentionClient, MentionError

logger = logging.getLogger(__name__)
client = MentionClient(api_key="your_api_key")

try:
    mentions = client.mentions.list(alert_id="alert_id")
except MentionError as e:
    logger.error(
        "Mention API error occurred",
        extra={
            "error_type": type(e).__name__,
            "message": e.message,
            "details": e.details
        }
    )
    raise
```
